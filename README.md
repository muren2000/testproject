# PDF Compress — офлайн-сжатие больших PDF и презентаций

Инструмент для macOS и Windows, который уменьшает размер больших PDF-файлов,
а также презентаций PowerPoint (.pptx) и документов Word/Excel (.docx/.xlsx) —
**полностью офлайн**: файлы никуда не отправляются и не покидают компьютер.

Есть два интерфейса:

- **графический** (окно с кнопками) — для повседневного использования;
- **командная строка** — для скриптов и пакетной обработки.

## Как это работает

1. Растровые изображения (главный источник «веса» PDF) уменьшаются до
   разумного разрешения и пересжимаются в JPEG с настраиваемым качеством.
2. Внутренняя структура PDF оптимизируется: объектные потоки, пересжатие
   flate-потоков, удаление неиспользуемых объектов и миниатюр страниц.
3. По умолчанию удаляются метаданные документа (можно отключить).

Для .pptx/.docx/.xlsx логика та же: это ZIP-архивы, изображения внутри
уменьшаются и пересжимаются (JPEG остаётся JPEG, PNG — PNG, прозрачность
сохраняется), архив переупаковывается с максимальным сжатием. Структура
документа и связи между частями не меняются.

Инструмент никогда не портит файл «в минус»: если сжатая версия вышла бы
больше исходной, на выходе будет копия исходника. Исходный файл не изменяется —
результат пишется в новый файл `имя.compressed.pdf` (`.compressed.pptx` и т.д.).

Типичный результат: сканы и презентации ужимаются на **60–90 %**.

## Уровни сжатия

| Пресет     | Назначение                              | Изображения        |
|------------|------------------------------------------|--------------------|
| `screen`   | Максимальное сжатие, чтение с экрана     | до 1200 px, JPEG 45 |
| `ebook`    | По умолчанию: планшеты, электронные книги | до 1800 px, JPEG 65 |
| `print`    | Умеренное сжатие, пригодно для печати    | до 2600 px, JPEG 82 |
| `lossless` | Без потерь: только структурная оптимизация | не изменяются     |

## Замеры на больших файлах

Синтетические, но реалистичные документы 60–65 МБ (Python 3.11, один поток):

| Файл                          | Пресет   | Было    | Стало   | Сжатие | Время | Пик памяти |
|-------------------------------|----------|---------|---------|--------|-------|------------|
| Фотоальбом, 38 стр., JPEG     | `screen` | 62.1 МБ | 2.5 МБ  | −96 %  | 8 с   | ~140 МБ    |
| Фотоальбом, 38 стр., JPEG     | `ebook`  | 62.1 МБ | 6.8 МБ  | −89 %  | 10 с  | ~140 МБ    |
| Фотоальбом, 38 стр., JPEG     | `print`  | 62.1 МБ | 17.9 МБ | −71 %  | 12 с  | ~140 МБ    |
| Скан A4 300 dpi, 13 стр., flate | `screen` | 65.3 МБ | 1.4 МБ  | −98 %  | 2 с   | ~140 МБ    |
| Скан A4 300 dpi, 13 стр., flate | `ebook`  | 65.3 МБ | 5.4 МБ  | −92 %  | 2 с   | ~140 МБ    |
| Скан A4 300 dpi, 13 стр., flate | `print`  | 65.3 МБ | 19.2 МБ | −71 %  | 3 с   | ~140 МБ    |

Память почти не зависит от размера файла — страницы обрабатываются по одной,
поэтому инструмент спокойно переваривает и файлы в сотни мегабайт.

## Установка (нужен Python 3.9+)

Python уже есть на macOS; на Windows поставьте его с [python.org](https://www.python.org/downloads/)
(при установке отметьте «Add Python to PATH»).

```bash
pip install pikepdf pillow
```

Дальше запускайте прямо из папки проекта — установка пакета не обязательна:

```bash
# Графический интерфейс
python -m pdfcompress --gui

# Командная строка
python -m pdfcompress документ.pdf                 # -> документ.compressed.pdf
python -m pdfcompress документ.pdf -p screen       # максимальное сжатие
python -m pdfcompress документ.pdf -o маленький.pdf
python -m pdfcompress *.pdf                        # пакетная обработка
python -m pdfcompress секрет.pdf --password 1234   # зашифрованный PDF
python -m pdfcompress документ.pdf --keep-metadata # не удалять метаданные
python -m pdfcompress презентация.pptx             # PowerPoint
python -m pdfcompress отчёт.docx таблица.xlsx      # Word и Excel
```

(на macOS вместо `python` может понадобиться `python3`)

Либо установите как пакет — появятся команды `pdfcompress` и `pdfcompress-gui`:

```bash
pip install .
```

## Автономные приложения (без Python у пользователя)

Скрипты собирают самодостаточные приложения через PyInstaller.
Сборка выполняется на целевой ОС (PyInstaller не кросс-компилирует).

**macOS** — получите `dist/PDF Compress.app` и CLI `dist/pdfcompress`:

```bash
bash scripts/build-macos.sh
```

**Windows** — получите `dist\PDF Compress.exe` и `dist\pdfcompress.exe`:

```bat
scripts\build-windows.bat
```

## Ограничения

- Старые бинарные форматы Office (.ppt, .doc, .xls — до Office 2007) не
  поддерживаются: их нельзя безопасно перепаковать без Microsoft Office.
  Откройте такой файл и пересохраните как .pptx/.docx/.xlsx.
- В офисных файлах метаданные не удаляются, а видео и другие не-графические
  вложения не пересжимаются.
- Изображения в форматах CCITT/JBIG2/JPEG2000 не перекодируются (они и так
  компактны либо перекодирование рискованно) — но структурная оптимизация
  применяется ко всему файлу.
- Для зашифрованных PDF нужен пароль (`--password`); результат сохраняется
  без шифрования.
- PDF, состоящие в основном из текста и векторной графики, сжимаются слабее —
  в них почти нечего «ужимать» без потери содержимого.
